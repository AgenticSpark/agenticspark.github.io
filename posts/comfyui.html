ComfyUI · Style-LoRA matrix

LoRA style matrix: 69 styles, one prompt loop

207 ComfyUI outputs — same three prompts, same seed, 69 style LoRAs. The methodology and every frame.

Tool
ComfyUI
LoRAs
69 styles
Outputs
207 images · 3 axes
Seed
42 (fixed)
Published
Nov 7, 2025

The standard way to showcase a style LoRA is one cherry-picked image. That tells you very little. This run fixed three prompts and one seed, then pushed every LoRA through the same pipeline. What you get is a fair comparison: the style's fingerprint across subject matter, not its best-case single frame.

69 styles. Three axes each. All outputs are below, grouped roughly by aesthetic family.

The three axes

  • Character. A single subject, eye-level, neutral background. Tests how a style renders skin, hair, fabric.
  • Landscape. Wide composition, no humans. Tests color palette, atmosphere, light handling.
  • Creative. A deliberately weird prompt — surreal composition, contradictory adjectives. Tests where the style breaks down.

Three images per style, sixty-nine styles, two hundred and seven outputs — all below.

What surprised me

The strongest styles aren't the ones that look most distinctive on the character axis — those usually just impose a fixed face. The strongest styles are the ones whose landscape and creative outputs still feel coherent: same palette, same rendering logic, even when the subject is unfamiliar. ghibli, wes_anderson, and klimt_gold all pass that bar. ukiyo_e nearly does but collapses on photorealistic subjects.

The loop

The whole matrix is driven by a ComfyUI service skill plus a small text2image dispatcher. Pseudocode: 

for style in styles:
    for axis in ["character", "landscape", "creative"]:
        prompt = PROMPTS[axis].format(style=style)
        ensure_comfyui_running()
        generate(prompt, lora=style, seed=42, out=f"{style}/{axis}.png")

Same seed across the matrix isn't strictly fair — some LoRAs respond very differently to a given seed — but it keeps the test reproducible, which matters more.

What I'd do differently next time

  • Add a fourth axis: text/UI. Most LoRAs cannot render type, and you only find out the hard way.
  • Run three seeds per axis and pick the median, not the first. The seed-42 bias is real.
  • Record VRAM and time per generation. Some "lightweight" LoRAs are anything but.
