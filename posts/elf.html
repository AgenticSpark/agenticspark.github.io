Paper · arXiv 2605.10938 · Diffusion language models

ELF · Embedded Language Flows

A continuous diffusion language model built on Flow Matching that stays continuous all the way to the final step — and only there does it argmax. One shared-weight network handles both the denoise (t<1) and the decode (t=1) branches, conditioned on a small mode token. The result: a 105M-parameter model that beats discrete diffusion baselines (MDLM, Duo) and prior continuous-DLMs (Diffusion-LM, FLM, LangFlow) using ~10× fewer training tokens — no distillation, no separate decoder, no per-step rounding loss.

ELF architecture: rectified-flow denoise in embedding space, then a final shared-weight decode step to vocabulary tokens.The same network draws the continuous flow toward the data embedding, then — once — projects to vocabulary logits and samples.

01 What

What is ELF?

A continuous diffusion language model. ELF (Embedded Language Flows) builds a Flow-Matching generator over embedding space, then uses the same network to perform a single discrete-decode step at the end. The thesis is sharp: most of the trouble previous continuous-DLMs had wasn't fundamental — it was algorithmic. Take rectified flow seriously, share weights, and drop the rounding/anchor losses, and the gap closes.

Continuous all the way

Every intermediate step lives in continuous embedding space. There is no per-step rounding to vocabulary, no simplex projection, no token-level anchor loss. The model only meets discrete tokens at t = 1.

Shared-weight final-step decode

One network xθ trained with x-prediction handles both the continuous denoiser (all t<1) and the discrete decoder (t=1), switched by a one-bit "mode" token. No separate decoder. Two losses, one set of parameters.

Image-domain tricks transfer cleanly

Rectified flow, x-prediction, self-conditioning, training-time Classifier-Free Guidance — every modern trick from continuous image diffusion ports in unchanged. CFG runs in a single forward pass at inference, with no double-evaluation overhead.

Where ELF sits among diffusion language models

Two big families: discrete (mask / categorical noise) and continuous (Gaussian noise in some embedding or latent). ELF is firmly in the continuous camp, and explicitly disagrees with the implicit consensus that discrete DLMs are the only ones worth pursuing for text.

02 How

How does it work?

Five moving pieces: an embedding map, a rectified-flow interpolant, an x-prediction objective, a shared-weight network that handles both denoise and decode, and a final-step cross-entropy. Plus two practical add-ons that ELF inherits from image diffusion — self-conditioning and training-time CFG.

2.1 · From discrete tokens to a continuous flow

Start with a sequence of token IDs s = (s1, …, sL) ∈ VL. An encoder E: V → ℝd embeds them into a continuous target x ∈ ℝL×d. ELF's headline choice: use the pretrained T5 encoder as E. (Ablations also try jointly-trained and frozen-random encoders; T5 wins.) The encoder is needed only at training time — never at inference.

Then build a rectified-flow interpolant between Gaussian noise ε ∼ 𝒩(0, I) and the data embedding x:

Rectified-flow interpolant · ε at t=0, data at t=1zt = t · x + (1 − t) · ε ∈ ℝL×d v = dzt / dt = x − ε

This is the standard rectified-flow / Flow-Matching setup. Sampling at inference will solve dz/dt = vθ(zt, t) from z0 = ε to z1 with an Euler ODE.

2.2 · x-prediction, not v-prediction · the load-bearing choice

Naive flow-matching parameterises a network vθ(zt, t) to predict the velocity directly. ELF instead parameterises xθ(zt, t) — the network predicts the data at every step. Velocity is recovered algebraically from the interpolant:

x-prediction · same MSE loss, target is the data embeddingvθ(zt, t) = ( zt − (1 − t) · xθ(zt, t) ) / t ℒMSE = 𝔼t, ε, x [ ‖ vθ(zt, t) − v ‖² ] = 𝔼 [ 1/(1−t)² · ‖ xθ(zt, t) − x ‖² ]

Why does this matter? Because at t → 1, the same network is asked to return the clean data embedding. That is the natural place to also ask: "now project to discrete tokens." x-prediction makes the shared-weight decode step possible without adding any extra heads.

2.3 · Two losses, one network · the shared-weight decode

The final step is the only step that meets discrete vocabulary. ELF gives the network a one-bit mode token (or equivalently runs t=1 through a special "decode" forward pass), and adds a cross-entropy loss on the vocabulary logits produced by an unembedding matrix W:

Final-step cross-entropy · unembedding projects x̂ to vocab logitspθ(s | z̃) = softmax( W · xθ(z̃, 1, decode) ) ℒCE = 𝔼z̃ [ −Σi log pθ(si | z̃) ] ℒ = ℒMSE + λ · ℒCE

The MSE loss trains the denoiser across all t ∈ (0, 1). The CE loss trains the decoder branch at t = 1. Both gradients flow into the same set of parameters; the mode token tells the network which job it's doing.

This is the single most important design decision. Prior continuous-DLMs train a separate decoder (often a separate softmax over the vocabulary applied to every intermediate step) and pay for it with both extra parameters and the dreaded "rounding-then-continuing" drift. ELF pays neither cost.

2.4 · Training loop · one batch, branch random

# Inputs:
#   tokens s, encoder E (T5), generator x_theta, unembedding W

x = E(s)                                     # L × d, continuous target
t = sample_time()                            # Beta-shifted toward t=1
eps = randn_like(x)
z_t = t * x + (1 - t) * eps                  # interpolant

mode = bernoulli(p=p_decode_branch)          # 0 = denoise, 1 = decode

if mode == 0:                                # --- denoise branch ---
    x_hat = x_theta(z_t, t, mode=denoise)
    v_hat = (z_t - (1 - t) * x_hat) / t
    v     = x - eps
    loss  = mse_loss(v_hat, v)

else:                                        # --- decode branch ---
    z1_tilde = perturb(x, sigma_small)       # near-data sample
    x_hat    = x_theta(z1_tilde, 1.0, mode=decode)
    logits   = z1_tilde @ W                  # or x_hat @ W (Eq. 9)
    loss     = cross_entropy(logits, s)

loss.backward(); optimizer.step()

Both branches share xθ. The decode branch sees a near-data sample z̃1 (a small perturbation of x) so its training distribution matches what the ODE solver actually produces at t = 1. This is the "train on samples that look like inference outputs" trick that latent diffusion models use, here applied to the text-decode boundary.

2.5 · Two practical add-ons

Self-conditioning · 50% of the time, free

In half the training steps, the network is given its own previous x-prediction as an additional input. At inference, the previous step's prediction is naturally available — so self-conditioning is essentially free per-step compute. Ablations show a consistent perplexity improvement at every step budget.

Training-time CFG · single forward pass at inference

Classical CFG runs two forward passes at inference (conditional + unconditional), doubling cost. ELF folds CFG into the training loss directly: each batch mixes conditional and unconditional samples in a controlled ratio, with the guidance scale conditioned in. At inference, one forward pass produces the CFG-guided velocity — no overhead. Cleanly transferred from image diffusion (Karras et al style training-time CFG).

2.6 · Inference · Euler ODE then a single argmax

An optional SDE-inspired sampler injects small noise at intermediate steps for variance control — borrowed from image-diffusion stochastic samplers, no algorithmic changes to the network. At every step except the last, the model lives in continuous space. The discrete world only shows up once.

03 Why

Why does it matter?

ELF makes a thesis-claim — "the continuous-DLM gap was algorithmic, not intrinsic" — and then proves it with strong, parameter-efficient, sample-efficient results across language modelling and conditional text tasks.

3.1 · The thesis

Continuous DLMs lost the race for fixable reasons

By 2024–25, the community had largely concluded that discrete diffusion LMs (MDLM, SEDD, Duo) were the right answer for text, and continuous DLMs (Diffusion-LM, CDCD, DiffuSeq) were a dead end. ELF rewinds: those continuous models were burdened with per-step rounding losses, simplex constraints, separate decoders, anchor losses and double-pass CFG. Each of those was a workaround for the absence of x-prediction + shared-weight decode. Remove them, do the basics right, and the gap closes.

3.2 · Empirical results

Generative perplexity per NFE

The paper's Figure 1 is the headline plot: ELF dominates MDLM, Duo, FLM, and LangFlow across the entire range of inference steps (NFEs from 8 to 1024). At every budget, ELF reaches a lower perplexity. The advantage is widest at low NFE — exactly where continuous DLMs were thought to be weakest.

Conditional generation · WMT'14 translation + XSum summarization

ELF outperforms the same baselines on both tasks, with the gap most visible at low NFE. The shared-weight decode preserves the conditioning signal across the entire flow — no "rounding drift" away from the conditioning that has plagued separate-decoder continuous DLMs.

Sample efficiency · 10× fewer training tokens

The MDLM / Duo baselines were trained for roughly 1 trillion tokens; ELF achieves better generative perplexity with ~100B tokens. Same model class, same data class — a 10× reduction in training compute. That's the kind of efficiency gain that justifies a new training paradigm.

3.3 · Ablation lessons

x-prediction > v-prediction

Switching the same architecture from v-prediction to x-prediction yields a clear perplexity drop. The shared-decode story only works with x-prediction; both improvements come bundled.

T5 encoder > jointly trained > frozen random

Same finding as in latent image diffusion: pretrained features make a better target space. Jointly training the encoder also works but adds optimisation friction. Frozen-random encoders are the floor.

Self-conditioning + training-time CFG ≈ free wins

Both add no inference compute. Both improve perplexity-per-NFE. Both are direct ports from image diffusion — the strongest single signal that the language/image divide is narrower than people thought.

A neat alignment with "Back to Basics"

ELF leans on the same minimalist principle as "Back to Basics" (the latent-diffusion-denoise paper that some of the same authors built): if you give a denoising objective a strong feature space to denoise in, you don't need rounding, simplex constraints, or per-step token losses. You just need to denoise. ELF carries that conviction from image latents to language embeddings.

3.4 · Honest limits

What it nails

  • Parameter-efficient — 105M beats 170M MDLM/Duo baselines at every NFE budget.
  • Sample-efficient — ~10× fewer training tokens than competing continuous DLMs.
  • Shared-weight decode — no separate decoder network, no rounding losses, no anchor losses.
  • Single-forward-pass CFG at inference. No double-evaluation overhead.
  • Image-domain methodology transfers cleanly — rectified flow, x-prediction, self-conditioning. The bridge is structural, not just metaphorical.
  • Conditional tasks improve in lockstep with unconditional language modelling — same gains on WMT'14 MT and XSum summarization.

Where it bends

  • Still trails autoregressive LMs at the high end of training compute. Diffusion-LM-as-foundation-model remains an open question at GPT-scale token budgets.
  • Sequence length is fixed up front — like all continuous-embedding DLMs, you commit to a length L at sample-start. Variable-length generation requires special tricks (e.g., a length predictor or padding+EOS).
  • Requires a strong encoder. T5 won the ablation; pre-existing high-quality text encoders are a hard requirement for the best results.
  • Final-step branching is bespoke. The mode-token / dual-loss training is one more lever to tune; not a plug-and-play replacement for either a pure flow-matching model or a pure autoregressive LM.
  • Single-language demo so far on standard English benchmarks. Multilingual and code-generation results aren't in this v1.

3.5 · Where this could go

Continuous-diffusion code / math models. Code and math have structured global constraints (balanced brackets, consistent variable use) that autoregressive LMs handle one-token-at-a-time. A continuous DLM with end-to-end revision capability and a shared-weight decode is a natural fit.

Multimodal natively. Text-image-audio in a single rectified-flow framework with three modality-specific encoders, one shared denoiser, three modality-specific decode branches. ELF's shared-weight pattern generalises with no friction.

Editing as inference. Continuous-DLMs can locally re-denoise spans of an existing embedding without re-decoding everything else. The shared-weight decode lets you commit only the spans you change. Useful for fine-grained text editing and constrained generation.

RLHF-friendliness. The single-pass-CFG conditioning extends naturally to reward signals. Once you have a continuous flow, you can incorporate reward-weighted velocity adjustments cleanly — without the discrete-action credit-assignment problems of autoregressive PPO.

The thing I'd want to see next: scale ELF to the 1B–3B parameter regime, and check whether the perplexity-per-NFE advantage holds. If it does, continuous DLMs become a credible parallel track to autoregressive language modelling — not as a research curiosity, but as a serious deployment option for tasks where revision, multimodality or controllability matters more than raw next-token throughput.
